# 云函数 openid 获取问题排查指南

## 问题现象
云函数 login 调用成功，但返回 `openid: undefined`，系统降级使用临时标识。

## 排查步骤

### 1. 查看云函数日志

**在微信开发者工具中查看：**
1. 打开微信开发者工具
2. 点击左侧「云开发」按钮
3. 进入「云函数」→ 选择 `login` 云函数
4. 点击「日志」选项卡
5. 查看最新的调用日志

**关键日志信息：**
- `login 云函数调用开始，event:` - 确认是否传入了 `code`
- `getWXContext 返回:` - 查看上下文信息
- `环境变量 APP_SECRET 是否存在:` - 确认环境变量是否读取成功
- `code2Session 响应:` - 查看微信 API 返回的具体错误

### 2. 确认环境变量配置位置

**重要：微信小程序云开发的环境变量需要在微信开发者工具中配置，不是在腾讯云 CloudBase 控制台！**

**正确配置步骤：**
1. 打开微信开发者工具
2. 点击左侧「云开发」按钮
3. 进入「云函数」→ 选择 `login` 云函数
4. 点击「配置」选项卡
5. 在「环境变量」部分添加：
   - **键名**：`APP_SECRET`
   - **值**：你的小程序 AppSecret（从微信公众平台获取）
6. 点击「保存」
7. **重要：保存后必须重新部署云函数！**

### 3. 重新部署云函数

配置环境变量后，必须重新部署：
1. 右键点击 `cloudfunctions/login` 文件夹
2. 选择「上传并部署（云端安装依赖）」
3. 等待部署完成

### 4. 验证 code 传递

在小程序端代码中，确认 `code` 已正确传递：
```javascript
wx.cloud.callFunction({
  name: 'login',
  data: {
    code: code  // 确保这里传入了 wx.login() 返回的 code
  },
  ...
})
```

### 5. 常见错误及解决方案

#### 错误：`环境变量 APP_SECRET 是否存在: false`

**原因：** 环境变量未正确配置或未重新部署

**解决：**
1. 确认在微信开发者工具的云函数配置中添加了 `APP_SECRET`
2. 确认环境变量名称完全一致（区分大小写）
3. 重新部署云函数

#### 错误：`code2Session 失败: errcode=40013, errmsg=invalid appid`

**原因：** AppID 不匹配

**解决：** 确认小程序 AppID 与云开发环境匹配

#### 错误：`code2Session 失败: errcode=40163, errmsg=code been used`

**原因：** code 已使用过（code 只能使用一次）

**解决：** 确保每次调用都使用新的 `wx.login()` 返回的 code

#### 错误：`code2Session 失败: errcode=40125, errmsg=invalid secret`

**原因：** AppSecret 错误

**解决：** 
1. 确认 AppSecret 正确（从微信公众平台重新生成）
2. 确认环境变量值没有多余空格
3. 重新部署云函数

### 6. 测试步骤

1. 在小程序中触发登录流程
2. 查看云函数日志，确认：
   - `event.code` 有值
   - `环境变量 APP_SECRET 是否存在: true`
   - `code2Session 成功，返回 openid:` 有值
3. 如果仍有问题，将完整的云函数日志截图或复制，便于进一步排查

## 注意事项

- 环境变量配置后必须重新部署云函数才能生效
- AppSecret 是敏感信息，不要提交到代码仓库
- code 只能使用一次，每次登录需要调用新的 `wx.login()`
- 如果使用腾讯云 CloudBase 控制台配置，可能不会生效，需要在微信开发者工具中配置
