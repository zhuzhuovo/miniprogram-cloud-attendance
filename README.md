# 公司员工打卡小程序

## 项目简介
这是一个供公司内部员工使用的极简打卡工具，核心是基于地理位置校验的上下班打卡，适合新手学习和快速部署。

## 技术栈
- 前端：微信小程序原生语法（JavaScript）
- 后端：微信小程序云开发（云数据库、云函数）
- 核心依赖：仅使用微信官方 API，不引入第三方框架

## 核心功能
1. **首页（打卡页）**
   - 自动获取用户微信授权（位置信息、用户信息）
   - 调用 `wx.getLocation` 获取实时定位
   - 校验定位是否在公司范围内（可配置公司经纬度和允许偏差半径）
   - 显示当前时间、打卡状态
   - 提供「上班打卡」/「下班打卡」按钮
   - 错误处理：位置权限拒绝、不在打卡范围内、网络异常时显示友好提示

2. **打卡历史页**
   - 展示用户所有打卡记录
   - 支持按日期筛选（今日/本周/本月）
   - 空状态提示（无打卡记录时显示引导文案）

3. **个人设置页**
   - 显示用户头像、昵称
   - 提供「修改打卡范围」入口（仅管理员可见）
   - 「清除本地缓存」功能

## 云数据库设计

### 1. 集合名称：`checkInRecords`

### 2. 字段说明
| 字段名 | 类型 | 说明 | 是否必填 |
|-------|------|------|---------|
| `_id` | String | 记录ID（自动生成） | 否 |
| `_openid` | String | 用户唯一标识（自动生成） | 否 |
| `type` | String | 打卡类型：'on' 上班 / 'off' 下班 | 是 |
| `checkInTime` | Number | 打卡时间（时间戳） | 是 |
| `location` | Object | 打卡经纬度：{lat: number, lng: number} | 是 |
| `status` | String | 状态：'normal' 正常 / 'late' 迟到 / 'early' 早退 | 是 |
| `createdAt` | Date | 记录创建时间（自动生成） | 否 |

### 3. 权限配置
- **读取权限**：仅创建者可读取（`{"read": "auth.openid == doc._openid"}`）
- **写入权限**：仅创建者可写入（`{"write": "auth.openid == doc._openid"}`）

## 部署步骤

### 1. 准备工作
- 注册微信小程序账号并获取 AppID
- 下载并安装微信开发者工具

### 2. 初始化项目
1. 打开微信开发者工具，选择「小程序」→「导入项目」
2. 填写项目信息：
   - 项目名称：公司员工打卡小程序
   - 目录：选择本项目文件夹
   - AppID：填写你的小程序 AppID
   - 开发模式：小程序
   - 后端服务：微信云开发

### 3. 配置云开发环境
1. 在微信开发者工具中，点击「云开发」按钮，开通云开发服务
2. 创建云开发环境，获取环境 ID
3. 修改 `app.js` 文件中的云开发环境 ID：
   ```javascript
   wx.cloud.init({
     env: '你的云开发环境ID', // 替换为你的云开发环境ID
     traceUser: true
   })
   ```

### 4. 部署云函数
1. 在微信开发者工具中，右键点击 `cloudfunctions/checkIn` 文件夹
2. 选择「上传并部署」→「云端安装依赖」

### 5. 创建数据库集合
1. 在云开发控制台中，选择「数据库」
2. 点击「+ 新建集合」，创建 `checkInRecords` 集合
3. 配置集合权限：选择「仅创建者可读写」

### 6. 配置公司位置
修改 `app.js` 文件中的公司位置配置：
```javascript
companyLocation: {
  lat: 31.2304, // 公司纬度
  lng: 121.4737, // 公司经度
  radius: 500 // 允许偏差半径（米）
}
```

### 7. 预览和发布
1. 在微信开发者工具中，点击「预览」按钮，使用微信扫码预览
2. 测试无误后，点击「上传」按钮，提交审核
3. 审核通过后，在小程序管理后台发布

## 新手常见问题排查

### 1. 云函数部署失败
- **问题**：云函数上传失败或部署失败
- **解决方案**：
  - 检查网络连接是否正常
  - 确保云开发环境已正确配置
  - 检查 `package.json` 文件中的依赖是否正确
  - 尝试重新上传并部署，选择「云端安装依赖」

### 2. 位置权限不生效
- **问题**：无法获取位置信息或位置权限被拒绝
- **解决方案**：
  - 在小程序管理后台，配置「位置权限」申请说明
  - 确保用户已授权位置权限
  - 检查手机定位服务是否开启

### 3. 不在打卡范围内
- **问题**：明明在公司却提示「不在打卡范围内」
- **解决方案**：
  - 检查 `app.js` 中的公司经纬度配置是否正确
  - 适当调整 `radius` 值（建议 300-500 米）
  - 检查手机定位精度是否足够（建议开启高精度定位）

### 4. 打卡记录不显示
- **问题**：打卡成功后，历史记录页不显示记录
- **解决方案**：
  - 检查云数据库 `checkInRecords` 集合是否有数据
  - 确保数据库权限配置正确（仅创建者可读写）
  - 检查网络连接是否正常

### 5. 云开发环境配置错误
- **问题**：调用云函数时提示环境错误
- **解决方案**：
  - 确保 `app.js` 中的云开发环境 ID 正确
  - 检查云函数的环境配置是否与小程序一致
  - 在微信开发者工具中，点击「云开发」→「设置」，查看环境信息

## 注意事项
1. 本项目使用微信小程序云开发，无需自建服务器
2. 请确保在小程序管理后台配置正确的权限说明
3. 公司位置信息建议在 `app.js` 中统一配置，方便后续修改
4. 实际部署前，请根据公司实际情况调整打卡时间和范围设置
5. 本项目仅供学习和内部使用，请勿用于生产环境
