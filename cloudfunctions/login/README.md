# login 云函数部署说明

## 若「上传并部署（云端安装依赖）」报错

### 方式一：先本地安装依赖，再上传（推荐）

1. 在电脑上打开终端，进入本目录：
   ```bash
   cd cloudfunctions/login
   ```
2. 执行安装依赖：
   ```bash
   npm install
   ```
3. 回到微信开发者工具，右键 `cloudfunctions/login`，选择 **「上传并部署」**（不要选「上传并部署：云端安装依赖」）。
4. 这样会直接上传已安装好的 `node_modules`，不再在云端执行 npm。

### 方式二：检查网络与镜像

云端安装依赖失败常见原因：网络超时或 npm 源在国外。

- 在微信开发者工具中：设置 → 代理设置，确认网络可用。
- 若使用自定义 npm 镜像，可在本目录新建 `.npmrc`，内容例如：
  ```ini
  registry=https://registry.npmmirror.com
  ```

### 方式三：精简后再试

已对 `package.json` 做了精简（固定版本、去掉多余字段）。若仍失败，请把报错**完整截图或复制**（含“更新云函数”/“安装依赖”阶段的提示），便于进一步排查。
